# Estilo de redação dos documentos

Simples não é pobre.
Simples é escrever de forma completa, com frases inteiras, e deixar a página visualmente agradável.

Vale para todo documento que uma pessoa lê: README, guias, opinião de uso, status.

## Os passos

1. Escreva a frase inteira, com artigo e conectivo.
   Prefira "Esta é a forma de usar com a IA" a "uso com a IA (a forma)".

2. Não use parêntese para repetir o que a frase já diz.
   Corte glosas como "(honesto)", "(a receita)", "(o difícil)".
   Se a palavra acrescenta, ela vira uma frase. Se não acrescenta, ela sai.

3. Não emende orações com travessão.
   Em vez de "ele não supera sozinho — nisso, use o topo", escreva duas frases.
   "Sozinho, ele não supera a competência pura. Para isso, use o topo."

4. Para separar ideias do mesmo assunto, quebre a linha.
   A quebra de linha é mais econômica e mais consistente do que emendar com travessão.
   Pode manter o mesmo parágrafo, com uma ideia por linha.

5. Use conectivos de verdade.
   São eles: porque, então, mas, quando, para que, já que, ou seja.
   Não deixe a pontuação fazer o trabalho do conectivo.

6. Use parêntese só para o que é mesmo acessório.
   Por exemplo: uma fonte, um número, um exemplo curto.
   Nunca para reforçar uma palavra que já está clara.

## Antes e depois

- "Como usar com IA (a receita)"
  vira "Como usar com a IA".

- "Funciona até com IA econômica — a forma é a alavanca."
  vira "Funciona até com a IA econômica, porque a forma é a alavanca."

- "só o topo, ou um humano no loop — nisso não supera"
  vira "Sozinho, ele não supera a competência pura. Para isso, use o topo, ou fique no loop."

## Onde isto não se aplica

O traço (o histórico) pode ser cronológico por natureza.
Tabelas e o frontmatter podem ser mais densos, porque ali a forma é de dado, não de prosa.
